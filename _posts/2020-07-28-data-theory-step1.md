## 1. 자료구조란?
---
```
"프로그램이란 데이터를 표현하고, 그렇게 표현된 데이터를 처리하는 것이다"
```
위에서 말하는 '데이터의 표현'은 '데이터의 저장'을 포함하는 개념입니다. 그리고 이렇듯 '데이터의 저장'을 담당하는 것이 바로 자료구조이고, 자료구조에서는 데이터를 표현하고 저장하는 방법에 대해서 설명합니다. 알고리즘은 표현이 저장된 데이터를 대상으로 하는 '문제의 해결 방법'을 뜻합니다.  
<br>

## 2. 시간 복잡도 & 공간 복잡도
---
## 알고리즘을 평가하는 두 가지 요소
+ 시간 복잡도(time complexity) => 얼마나 빠른가?  
+ 공간 복잡도(space complexity) => 얼마나 메모리를 적게 쓰는가?
+ 결론적으로 시간 복잡도를 더 중요시 한다.

+ __시간 복잡도를 평가하는 방법__
    - 중심이 되는 특정 연산의 횟수를 세어 평가한다.
    - 데이터의 수에 대한 연산횟수의 함수 T(n)을 구한다.

+ __알고리즘 수행 속도 비교 기준__
    - 데이터의 수가 적은 경우의 수행 속도는 큰 의미가 없다.
    - 데이터의 수에 따른 수행 속도의 변화 정도를 기준으로 한다.
<br>

## 3. 순차 탐색 알고리즘과 시간 복잡도
---
```
//순차 탐색 알고리즘이 적용된 함수
int LSearch(int ar[], int len, int target)
{
    itn i;
    for(i=0; i<len; i++)
    {
        if(ar[i] == target) return i; 찾은 대상의 인덱스 값 반환
    }
    return -1;      // 찾지 못했음을 의미하는 반환값
}
```
어떠한 연산자의 연산 횟수를 세어야 할까요 ? (==) 의 횟수를 세야합니다. 그렇다면 어떠한 상황에서의 시간 복잡도를 계산해야 할까요?  
+ 최상의 경우 : 맨 처음에 바로 대상을 찾는 경우
+ 최악의 경우 : 제일 마지막에 대상을 찾는 경우
+ 평균적인 경우 : 가장 현실적인 경우

정답은 최악의 경우입니다. 왜냐하면 만족스럽지 못한 상황에서의 성능평가가 주 관심사이기 때문입니다. 순차 탐색의 경우는 시간 복잡도를 간단하게 아래처럼 표현할 수 있습니다.
```
"데이터의 수가 n개일 때, 최악의 경우에 해당하는 연산 횟수는(비교연산의 횟수는) n이다."  
즉, T(n) = n 이다.
```

## 4. 이진 탐색 알고리즘의 소개
---
### 이진 탐색 알고리즘의 첫 번째 시도
target 값을 3으로 가정하고 설명을 진행하겠습니다.

1. 배열 인덱스의 시작과 끝은 각각 0과 8이다.
2. 0과 8을 합하여 그 결과를 2로 나눈다.
3. 2로 나눠서 얻은 결과 4를 인덱스 값으로 하여 arr[4]에 저장된 값이 3인지 확인한다.

![그림1](https://backtony.github.io/assets/img/post/data/step1/1.PNG)


### 이진 탐색 알고리즘의 두 번째 시도
1. arr[4]에 저장된 값이 9와 탐색 대상인 3의 대소를 비교한다.
2. 대소 비교결과는 arr[4] > 3 이므로 탐색 범위를 인덱스 기준 0~3으로 제한한다.
3. 0과 3을 더하여 그 결과를 2로 나눈다. 이때 나머지는 버린다.
4. 2로 나눠 얻은 결과가 1이니 arr[1]에 저장된 값이 3인지 확인한다.

![그림2](https://backtony.github.io/assets/img/post/data/step1/2.PNG)

### 이진 탐색 알고리즘의 세 번째 시도
1. arr[1]에 저장된 값 2와 탐색 대상인 3의 대소를 비교한다.
2. 대소 비교결과 arr[1] < 3 이므로 탐색의 범위를 인덱스 기준 2~3으로 제한한다.
3. 2와 3을 더하여 그 결과를 2로 나눈다. 나머지는 버린다.
4. 2로 나눠서 얻은 결과가 2이니 arr[2]에 저장된 값이 3인이 확인한다.

![그림3](https://backtony.github.io/assets/img/post/data/step1/3.PNG)

이처럼 매 탐색마다 대상을 반씩 줄여나가기 때문에 순차 탐색보다 비교적 좋은 성능을 보입니다. 그럼 이를 바탕으로 이제 코딩을 해봅시다.
```
int BSearch(int ar[], int len, int target)
{
    int first =0;       // 탐색 대상의 시작 인덱스 값
    int last = len-1;   // 탐색 대상의 마지막 인덱스 값
    int mid

    while(first <= last)
    {
        mid = (first+last)/2; // 탐색대상 중간
        if(target == ar[mid]) return mid;
        else
        {
            if(target < ar[mid]) last = mid-1;
            else first = mid +1;            
        }
    }
    return -1; // 찾지 못했을때 -1 반환
}
```
while문 조건은 first 와 last가 같을 때도 그 해당 값을 검사 해줘야 하므로 first와 last가 역전될때까지 탐색과정을 계속해야 합니다.  else 문에서 mid +1 또는 -1 을 해야하는 이유는 +1 또는 -1을 추가하지 않을 경우 탐색대상이 존재하지 않을때 first와 last가 역전하는 현상이 발생하지 않기 때문입니다.

## 5. 이진 탐색 알고리즘 최악의 경우 시간 복잡도
---
## 비교 연산의 횟수 예
+ 8이 1이 되기까지 2로 나눈 횟수 3회, 따라서 비교연산 3회 진행
+ 데이터가 1개 남았을 때, 마지막으로 비교연산 1회 진행

## 비교 연산의 횟수의 일반화
+ n이 1이 되기까지 2로 나눈 횟수 k회, 따라서 비교연산 k회 진행
+ 데이터가 1개 남았을 때, 마지막으로 비교연산 1회 진행.
+ 결과적으로 T(n) = k+1

즉, n * (1/2)^k = 1 로 식을 세울 수 있습니다.  
따라서, 정리하면 T(n) = log_n +1  
그러나 +1은 생략이 가능합니다. 이에 대한 이유는 빅-오 표기법에서 알 수 있습니다.  
<br>

## 6. 빅-오 표기법(Big-Oh Notation)
---
+ T(n)이 다항식으로 표현이 된 경우, 최고차항의 차수가 빅-오가 된다.
+ 최고차항의 계수는 무시한다.

예를 들면 T(n) = 10n^2 +2n +1 의 빅오는 O(n^2) 입니다.
